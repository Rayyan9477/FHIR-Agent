This directory contains the P0 Reconciliation Coordinator agent config.

## Files

| File              | Purpose                                                                |
|-------------------|------------------------------------------------------------------------|
| `system_prompt.md` | The system prompt — the **only** file the model actually reads        |
| `agent.yaml`      | Prompt Opinion A2A agent config (schema TBD per RISKS Q2)              |
| `README.md`       | This file                                                              |

## Marketplace publishing (Phase 9)

1. Sign in to https://app.promptopinion.ai
2. Marketplace settings → Add agent
3. Upload `agent.yaml` (or fill the form fields with its values)
4. Set the MCP server URL: the public HTTPS endpoint from `ngrok` /
   `cloudflared` (Phase 9 captures this in `.ngrok-url`)
5. Set capability tag: `medrec.reconcile`
6. Reference the output schema: `agents/schemas/reconciliation_report.json`
   (generated by `python scripts/export_schemas.py`)
7. Submit for marketplace review

## Local testing

For local agent testing without the marketplace:

```bash
# 1. Generate a SHARP test keypair + token
python -c "from tests.conftest import sharp_keypair  # private + public"

# 2. Set env to point at the local server
export SHARP_PUBLIC_KEY_PEM=./tests/fixtures/sharp/keypair_test.pub.pem
export ANTHROPIC_API_KEY=sk-ant-...

# 3. Start the MCP server
uv run python -m medrec_superpower

# 4. Run the e2e test scenario
uv run pytest tests/integration/test_p0_metformin.py -v
```

The integration test stands in for the live Coordinator until `ANTHROPIC_API_KEY`
and the Prompt Opinion runtime are available (Phase 10).

## Hard rules the system prompt enforces

These are documented in `system_prompt.md` and **mechanically enforced** by
the MCP server (the prompt is defense-in-depth, not the sole guarantee):

| # | Rule | Mechanically enforced by |
|---|------|--------------------------|
| R1 | `patient_id` from SHARP, never LLM | `@requires_sharp` decorator (V5) |
| R3 | Drug data from tools, never LLM | `check_interaction` `check_succeeded` flag |
| R5 | `status=hold` → no daily plan | `ReconciliationReport` model validator |

## Phasing

- **P0** (this file): Coordinator-only, Markdown report, inline safety verdict
- **P1**: + Patient Educator handoff for 6th-grade narrative
- **P2**: + Drug Safety Specialist handoff for binding `hold` verdicts
